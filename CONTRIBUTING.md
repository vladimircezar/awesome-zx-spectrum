# Contributing to Awesome ZX Spectrum

Thank you for helping improve this list. Contributions are welcome, and the core
rule is quality over quantity.

## What belongs here

Good additions are:

- clearly relevant to ZX Spectrum
- high quality
- durable
- useful
- unique enough to deserve a slot

## What usually does not belong here

These are usually out:

- generic retro gaming pages with almost no Spectrum relevance
- random store listings
- link farms
- dead projects with no historical significance
- duplicate resources
- personal pages with thin content
- direct piracy-focused resources

## Entry format

- Use `- [Name](URL) - Description.`
- Keep descriptions short.
- Do not oversell.
- Avoid marketing language.
- Prefer project home pages over third-party mirrors.
- Prefer the most canonical link.

## Placement rules

- Put items in the best-fitting section.
- Avoid duplicates across multiple sections unless there is a very good reason.
- Keep sections alphabetized where practical.

## Quality bar

Before submitting, verify all of the following:

- link works
- content is relevant
- content has substance
- content is not obviously spammy
- description is accurate
- project is still accessible, unless included for historical importance

## Historical resources

Historical or archived resources are welcome when they are important. If a
resource is old but significant, that is a feature, not a bug. Keep the
description honest about what the resource is today.

## Pull request checklist

Copy this checklist into your pull request description:

- [ ] I checked the link works.
- [ ] I checked the resource is relevant to ZX Spectrum.
- [ ] I checked it is not already listed.
- [ ] I wrote a short neutral description.
- [ ] I placed it in the best section.
- [ ] I sorted the section alphabetically where appropriate.

- tutorials
- blogs and newsletters
- books and long-form writing
