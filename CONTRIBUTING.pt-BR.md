# Contribuindo para Awesome ZX Spectrum

Obrigado por ajudar a melhorar esta lista. Contribuições são bem-vindas, e a
regra principal é qualidade acima de quantidade.

## O que pertence aqui

Boas adições são:

- claramente relevantes para ZX Spectrum
- de alta qualidade
- duráveis
- úteis
- únicas o bastante para merecer um espaço

## O que normalmente não pertence aqui

Estes itens normalmente ficam de fora:

- páginas genéricas de jogos retrô com quase nenhuma relevância para Spectrum
- listagens aleatórias de lojas
- fazendas de links
- projetos mortos sem importância histórica
- recursos duplicados
- páginas pessoais com pouco conteúdo
- recursos focados diretamente em pirataria

## Formato das entradas

- Use `- [Nome](URL) - Descrição.`
- Mantenha descrições curtas.
- Não exagere.
- Evite linguagem de marketing.
- Prefira páginas oficiais dos projetos a espelhos de terceiros.
- Prefira o link mais canônico.

## Regras de posicionamento

- Coloque itens na seção mais adequada.
- Evite duplicatas em múltiplas seções, salvo motivo muito forte.
- Mantenha seções em ordem alfabética quando fizer sentido.

## Critério de qualidade

Antes de enviar, verifique tudo abaixo:

- o link funciona
- o conteúdo é relevante
- o conteúdo tem substância
- o conteúdo não é obviamente spam
- a descrição é precisa
- o projeto continua acessível, salvo inclusão por importância histórica

## Recursos históricos

Recursos históricos ou arquivados são bem-vindos quando forem importantes. Se
um recurso é antigo, mas significativo, isso é uma qualidade. Apenas mantenha a
descrição honesta sobre o que ele é hoje.

## Checklist de pull request

Copie este checklist para a descrição do pull request:

- [ ] Eu verifiquei se o link funciona.
- [ ] Eu verifiquei se o recurso é relevante para ZX Spectrum.
- [ ] Eu verifiquei se ele ainda não está listado.
- [ ] Eu escrevi uma descrição curta e neutra.
- [ ] Eu coloquei o item na melhor seção.
- [ ] Eu ordenei a seção em ordem alfabética quando apropriado.
- [ ] Quando relevante, eu atualizei README.md e README.pt-BR.md.
